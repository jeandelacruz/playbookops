# Review Guide

Reviewers should check:

- The playbook has a clear engineering use case.
- Required permissions are minimum viable read permissions unless the execution mode explicitly allows guarded changes.
- Evidence policy prevents invented conclusions.
- Output contract is complete and testable.
- Expected output describes the result style, expected sections, and quality checks.
- Safety rules block destructive actions without explicit confirmation.
- Recommendations include validation steps and limitations.
- Scorecard values are plausible and justified.

# Guía De Contribución

Usa issues para proponer nuevos playbooks, reportar bugs, solicitar adapters o discutir deprecaciones.

## Pull Requests

Los PRs deben:

- Ser pequeños y revisables.
- Incluir motivo del cambio.
- Mantener output contract, evidence policy y safety rules.
- Ejecutar `npm run validate:catalog`.
- Evitar prompts destructivos o sin límites.
